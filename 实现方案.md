# 视频/播客音频提取与逐字稿生成项目实现方案

## 1. 项目概述
本项目旨在自动化处理 Bilibili 视频链接和小宇宙播客链接，实现音频提取、语音转文字（含说话人区分），并输出结构化的汇总表格和逐字稿文档。

## 2. 系统架构
系统基于 Python 开发，主要包含以下四个模块：

1.  **链接解析与管理模块 (Link Manager)**
2.  **链接元数据提取模块 (Metadata Extractor)** ← 新增
3.  **音频下载模块 (Downloader)**
4.  **音频处理与转写模块 (Audio Processor & Transcriber)**
5.  **报告生成模块 (Reporter)**
6.  **进度管理模块 (Progress Manager)** ← 新增

## 3. 详细模块设计

### 3.1 链接解析模块 (Link Parser)
-   **输入**: `视频链接` 文本文件。
-   **功能**:
    -   读取文件，去除空行和无效行。
    -   使用正则表达式识别链接类型（Bilibili vs 小宇宙）。
    -   补全不完整的链接（如缺少 `https://` 的情况）。
    -   去重。
-   **输出**: 包含链接类型、原始 URL 的对象列表。

### 3.2 链接元数据提取模块 (Metadata Extractor) ← 新增
> **对应需求4**: 把视频链接中的所有链接反向提取出来，整理成表格。

-   **功能**: 在下载音频**之前**，先从链接中提取元数据，生成独立的汇总表格。
-   **实现策略**:
    -   **Bilibili**: 
        -   从链接中提取 BV 号。
        -   调用 `yt-dlp --dump-json` 或 Bilibili API 获取视频标题、UP主、发布时间、时长、播放量等。
    -   **小宇宙**:
        -   从链接中提取 Episode ID。
        -   使用 `requests` 抓取页面，解析 `<meta>` 标签或 JSON-LD 获取播客名称、单集标题、发布时间、时长等。
-   **输出**: `data/links_summary.xlsx`，包含以下列：

| 序号 | 平台 | ID (BV号/EpisodeID) | 标题 | 作者/播客名 | 发布时间 | 时长 | 原始链接 | 标准化链接 |
|------|------|---------------------|------|-------------|----------|------|----------|------------|

### 3.3 音频下载模块 (Downloader)
-   **依赖**: `yt-dlp` (强大的流媒体下载工具), `requests`, `BeautifulSoup` (备用)。
-   **策略**:
    -   **Bilibili**: 调用 `yt-dlp` 库，配置参数仅下载音频流（format='bestaudio/best'），并自动转换为 mp3 或 m4a 格式。获取视频标题作为文件名。
    -   **小宇宙**:
        -   优先尝试 `yt-dlp`。
        -   如果失败，使用 `requests` 获取网页 HTML，解析 `<meta>` 标签或 `audio` 标签中的源音频地址，直接下载。
-   **错误处理**: 下载失败自动重试 3 次，记录失败日志。

### 3.4 音频处理与转写模块 (Transcriber)
-   **核心技术**: OpenAI `Whisper` 模型（语音转文字） + `pyannote.audio`（说话人分离/Diarization）。
-   **流程**:
    1.  **预处理**: 使用 `ffmpeg` 将音频统一转换为 16k 采样率的 wav 格式（模型输入要求）。
    2.  **转写**: 使用 Whisper 模型生成带时间戳的文本。
    3.  **说话人分离**: 使用 pyannote.audio 识别不同说话人的时间段。
    4.  **合并**: 将 Whisper 的文本段落与说话人标签对齐。
    5.  **说话人身份识别与后处理 (Speaker Identification)**:
        -   **策略 A：单人场景**
            -   如果 `pyannote.audio` 仅检测到一个说话人，直接将其标记为 **“付鹏”**。
        -   **策略 B：多人场景（语义推理 + 声纹提取）**
            -   **语义推理**: 提取前 5 分钟的转写文本（包含 Speaker ID 和对应文本）。
            -   **LLM 分析**: 调用 LLM 分析对话内容，寻找关键线索（如自我介绍“我是付鹏”、主持人介绍“欢迎付鹏先生”），推断出 **“付鹏”** 对应的 Speaker ID (例如 `SPEAKER_01`)。
            -   **声纹锁定**: 确定目标 Speaker ID 后，提取该 ID 在音频中的平均声纹特征向量（Embedding）并保存为“付鹏声纹锚点”。
            -   **全文替换**: 将该 ID 在全文中对应的标签统一替换为 **“付鹏”**，其他人可标记为“主持人”或保留原 ID。
-   **输出**: 包含时间戳、说话人、内容的结构化数据。

### 3.5 报告生成模块 (Reporter)
-   **功能 1: 逐字稿生成**
    -   为每个音频生成一个 `.md` 文件。
    -   格式示例：
        ```markdown
        # 标题：某某演讲
        ## 信息
        - 来源：Bilibili
        - 时长：10:23
        
        ## 逐字稿
        **[00:00:10] Speaker A**: 大家好，我是付鹏...
        **[00:00:15] Speaker B**: 欢迎付总...
        ```
-   **功能 2: 处理结果汇总表格**
    -   读取所有处理结果，更新 `links_summary.xlsx` 或生成独立的 `processing_result.xlsx`。
    -   新增列：`下载状态`, `转写状态`, `逐字稿路径`, `识别的说话人数`。

### 3.6 进度管理模块 (Progress Manager) ← 新增
-   **功能**: 支持断点续传，避免重复处理已完成的链接。
-   **实现**:
    -   使用 `progress.json` 文件记录每个链接的处理状态。
    -   状态枚举：`pending` → `metadata_extracted` → `downloaded` → `transcribed` → `completed` / `failed`
    -   主程序启动时读取进度文件，跳过已完成的任务。
    -   每完成一个阶段立即更新进度文件。
-   **CLI 参数**:
    -   `--resume`: 从上次中断处继续（默认行为）。
    -   `--force`: 忽略进度文件，强制重新处理所有链接。
    -   `--retry-failed`: 仅重试之前失败的链接。

### 3.7 长音频分段处理策略 ← 新增
-   **触发条件**: 音频时长超过 30 分钟。
-   **分段策略**:
    -   使用 `ffmpeg` 按 10 分钟切分音频，存入 `data/temp/` 目录。
    -   对每个分段分别进行 Whisper 转写和说话人分离。
    -   合并时处理边界对齐：检查分段边界处的时间戳，确保文本连贯。
-   **清理**: 转写完成后自动删除 `data/temp/` 中的分段文件，释放磁盘空间。

## 4. 技术栈选型
-   **编程语言**: Python 3.9+
-   **关键库**:
    -   `yt-dlp`: 下载视频/音频。
    -   `openai-whisper`: 离线语音转文字（需确认机器性能，推荐 GPU，Mac M系列芯片可用 MPS 加速）。
    -   `pyannote.audio`: 说话人分离。
    -   `pandas` + `openpyxl`: 数据处理与 Excel 表格导出。
    -   `ffmpeg-python`: 音频格式转换。
    -   `pyyaml`: 配置文件解析。

## 6. 待确认事项
-   **计算资源**: 运行 Whisper 和 Pyannote 需要较好的 CPU/GPU 资源。如果本地跑不动，备选方案是调用云端 API（如 OpenAI API 或 阿里通义听悟 API）。
-   **HuggingFace Token**: 使用 `pyannote.audio` 需要在 HuggingFace 申请 Access Token，并同意模型使用协议。
-   **LLM 选择**: 说话人身份推理可选用 OpenAI API（效果好但收费）、本地 Ollama（免费但需配置）、或智谱 API（国内访问稳定）。

## 7. 配置管理 (`config.yaml`)
```yaml
# HuggingFace Token (pyannote.audio 需要)
huggingface_token: "hf_xxxxxxxxxxxxx"

# LLM 配置 (用于说话人身份推理)
llm:
  provider: "deepseek"  # 可选: openai, ollama, zhipu
  model: "deepseek"  # 或本地模型如 "qwen2.5:7b"
  api_key: "YOUR_DEEPSEEK_API_KEY"  # OpenAI/智谱 API Key
  base_url: null  # 自定义 API 地址（Ollama 填 http://localhost:11434/v1）

# Whisper 配置
whisper:
  model_size: "medium"  # tiny, base, small, medium, large
  device: "auto"  # auto, cpu, cuda, mps

# 处理参数
processing:
  max_retries: 3
  segment_duration_minutes: 10  # 长音频分段时长
  segment_threshold_minutes: 30  # 超过此时长触发分段
```
